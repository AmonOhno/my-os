# 残す — 記録シート

## 1. 目的

**3タップ以内で記録が終わる**こと。開く(1) → 領域を選ぶ(2) → 書く/気分を選ぶ → のこす(3)。
続けられるかどうかはこの画面で決まる。迷わせる要素・必須項目を増やさない。

## 2. 起動

- どの画面からでも右下の FAB(+) でボトムシートとして開く。
- シートを閉じる操作（下スワイプ / 背景タップ）で下書きは破棄する。下書き保存は作らない（記録は軽いものという位置づけ）。

## 3. エントリの種類（kind）

| kind | 名称 | 入力UI | 保存内容 |
|------|------|--------|---------|
| `note` | メモ | 複数行テキスト | `body`（自由文） |
| `mood` | 気分 | 5段階ピッカー + 一言 | `mood`(1–5) + `body`（一言、任意） |
| `metric` | 数値 | 数値 + 単位 + ラベル | `payload: { label, value, unit }`（例: 体重 62.4 kg / 学習 1.5 h） |
| `event` | 出来事 | タイトル + 補足 | `title` + `body`（任意）。ライフイベント用 |

- シート最上部のセグメントで種類を切り替える。デフォルトは `note`。
- 種類を切り替えても入力済みの領域選択は維持する。

## 4. 入力項目

| 項目 | 必須 | 仕様 |
|------|------|------|
| 種類 | ○ | 上記4種。デフォルト `note` |
| 領域 | ○（1つ以上） | チップの複数選択。よく使う順に並べる（直近30日の使用頻度順、初期は Health, Career, Learning, Mental, …） |
| 本文/値 | kindによる | `note`: body必須 / `mood`: mood必須・body任意 / `metric`: label+value必須・unit任意 / `event`: title必須 |
| 日時 `occurredAt` | ○（自動） | デフォルトは現在時刻。「きのうのことを書く」ために日時変更ボタンだけ置く（カレンダーUIは簡素に） |

### metric の単位・ラベル

- 直前に使った label/unit の組をサジェストする（例: 一度「体重/kg」を記録したら次回は1タップで選べる）。
- サジェストは entries から導出する（別テーブルは作らない）。

## 5. バリデーション

- 必須項目が空のあいだは「のこす」ボタンを無効化する。エラーメッセージは出さない（押せないだけ）。
- `mood` は 1–5 の整数。`metric.value` は数値（負数も許可 — 収支メモなど）。
- 文字数制限は設けない。

## 6. 保存処理

1. `ent_<uuid>` を採番し、`entries` に保存（[schema.md](../database/schema.md)）。
2. シートを閉じ、元の画面に戻る。きょう画面ならタイムライン先頭に新エントリが現れる。
3. 保存失敗（IndexedDB例外）はシートを閉じずにトースト表示。入力は保持する。

## 7. しないこと

- ストリーク表示・「昨日書いていません」等の督促
- 下書きの自動保存・テンプレート・定型文
- 写真添付（必要になった時に payload 拡張で検討）
