# 見る — きょう（ホーム）

## 1. 目的

ホームは「きょうの自分」。今日〜直近のエントリが時系列で流れ、領域ごとの最新の状態が一枚で見える。
**グラフや達成率は出さない。** あくまで状態を「眺める」画面。

## 2. レイアウト（上から順）

UIMockUp のきょう画面に準拠。

1. **あいさつ + 今日の日付** — 「7月4日(土) こんにちは。」時間帯であいさつを変える（おはよう/こんにちは/こんばんは）。
2. **領域の状態カード**（横スクロール）
3. **タイムライン**（直近のエントリ）
4. 右下 FAB(+) → 記録シート / 下部タブバー

## 3. 領域の状態カード

- 各領域につき1枚。**直近のエントリがある領域のみ**表示する（空の領域のカードは出さない — 責めない設計）。
- カード内容: 領域名 + その領域の最新状態の要約1行 + 補足1行。

| 領域 | 要約に出すもの |
|------|---------------|
| Health / Learning など | 最新エントリの要約（metric なら `睡眠 6.5h`、note なら本文先頭） |
| Mental | 最新の mood（絵文字/ラベル + 一言） |
| Finance | 既存アプリの今月収支サマリ（`今月 +42,300円` / 取得元表記）。取得失敗時は「取得できませんでした」 |

- 並び順: 最新エントリが新しい領域順。Finance は常に先頭から2番目に固定（モック準拠、目に入るが主役にしない）。
- カードtap → その領域のタイムライン絞り込み（Finance のみ Finance領域ビュー → [finance_integration.md](finance_integration.md)）。

## 4. タイムライン

- `occurredAt` 降順。初期表示は直近14日分、スクロールで過去を追加読込（無限スクロール）。
- 各行: 領域チップ（複数可）+ 時刻 + 内容。
  - `note`: 本文（3行まで、以降省略）
  - `mood`: 気分アイコン + 一言
  - `metric`: `ラベル 値単位`
  - `event`: タイトルを強調表示
- 日付見出しでグルーピング（きょう / きのう / 7月2日(木) …）。
- **エントリのない日は見出し自体を出さない。**「記録なし」のような空行を作らない。
- 行tap → エントリ詳細（編集・削除・つなげる）。

## 5. 領域絞り込みビュー

- 状態カードtapで遷移。ヘッダーに領域名、以下は同じタイムラインをその領域のみでフィルタ。
- 戻るはヘッダー左の「< きょう」。

## 6. エントリ詳細

- 全項目の表示 + 編集（記録シートと同じフォームを再利用）・削除（確認ダイアログあり）。
- 「つなげる」ボタン → リンク作成フロー（[connections.md](connections.md) §3）。
- このエントリに既にあるつながりを下部にリスト表示。

## 7. しないこと

- 週間サマリ・グラフ・達成率・平均値
- 「今日はまだ記録がありません」のような催促文言（空のときは静かに空でよい。タイムラインが空なら直近のエントリだけ見せる）
