# 考える — つながり

## 1. 目的

エントリ同士を手動でつなげ、領域を横断した関係を「地図」として眺める。
「残業が続いた」と「眠りが浅い」をつなぐ、のように。**気づくのは自分**であり、アプリは関係を提示しない（AIの相関分析はしない — 要件定義書 §6）。

## 2. レイアウト（上から順）

1. **期間切替** — 月単位（`‹ 2026年7月 ›`）。表示対象はその月に `occurredAt` を持つエントリ + それらに繋がるリンク。
2. **つながりマップ** — エントリのノード + リンクの線。
3. **ふりかえりメモ** — その月のふりかえり（reviews）。
4. 下部タブバー。

## 3. リンク（つながり）の作成

- 起点は2つ:
  - **エントリ詳細**の「つなげる」→ 相手エントリを検索/最近のエントリから選択 → 任意の一言メモ（`note`）→ 保存。
  - **マップ上**でノードを長押し → もう1つのノードをtap → 一言メモ → 保存。
- リンクは**無向**として扱う（保存上は from/to を持つが、表示・重複判定では方向を無視する）。
- 同じ2エントリ間の重複リンクは作成不可。
- 削除はリンクの線 or 詳細画面のつながりリストから。

## 4. つながりマップ

- 描画は SVG + d3-force（ノード数は月あたり数十個想定）。
- **ノード**: 円 + 短いラベル（title または本文先頭)。色は主領域（エントリの最初の領域）の領域色。
- **リンク**: 細い線。一言メモがあれば線の中央に小さく表示。
- リンクを持たないエントリはマップに出さない（点の海にしない）。ただし「この月のエントリからつなぐ」導線をマップ下に置く。
- 操作: ピンチズーム / パン / ノードtap → エントリ詳細。
- レイアウトは開くたびに force simulation で計算（座標は保存しない。将来固定したくなったら payload でなく別テーブルを検討）。

## 5. 月次ふりかえり（reviews）

- 月にいちど、その月のエントリとつながりを見ながら**気づきを1本のメモ**として残す。
- マップ下の「ふりかえり」カード:
  - その月の review が未作成 → 「この月をふりかえる」ボタン（月末に限らずいつでも書ける。督促はしない）。
  - 作成済み → 本文を表示。tapで編集。
- 1月につき1レコード（`month` はユニーク）。書き直しは上書き編集。
- ふりかえり作成画面では、参考として同じ画面内にその月のマップのサムネイルとエントリ件数を添える。
- 保存時にさりげなく「エクスポートも忘れずに」の一文とエクスポートへのリンクを出す（バックアップ導線 — [architecture/overview.md](../architecture/overview.md) §6。通知ではなく画面内表示）。

## 6. しないこと

- 自動リンク・関連候補のサジェスト（AI領域のため設計しない）
- リンク数のカウント表示・「今月のつながり数」等の指標化
- 月をまたぐマップの統合ビュー（必要になったら設計する）
