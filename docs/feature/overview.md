# 機能一覧・画面遷移

## 1. 基本機能と画面の対応

基本機能は3つだけ。それぞれが1つの画面に対応する（ConceptSheet §03）。

| 機能 | 画面 | 概要 | 詳細 |
|------|------|------|------|
| 残す | 記録シート | 3タップ以内で状態を記録する | [record.md](record.md) |
| 見る | きょう（ホーム） | 今日〜直近の自分を眺める | [today.md](today.md) |
| 考える | つながり | エントリ同士の関係を見て理解する | [connections.md](connections.md) |

補助画面として **エントリ詳細**（編集・削除・リンク作成の起点）と **設定**（Finance トークン・エクスポート/インポート）を持つ。

## 2. 8つの領域（domain）

領域はフィルタであって壁ではない。1エントリが複数領域に属せる。

| キー | 表示名 | 対象 |
|------|--------|------|
| `health` | Health | 身体の状態 |
| `career` | Career | 仕事・スキル |
| `learning` | Learning | 学習・知識 |
| `family` | Family | 家族・ライフイベント |
| `finance` | Finance | お金（既存アプリの読み取り表示のみ） |
| `projects` | Projects | 個人開発・アイデア |
| `mental` | Mental | 気分・ストレス |
| `travel` | Travel | 旅・リフレッシュ |

`finance` はエントリの領域チップとしても選べる（例:「大きな買い物をした」というメモ）。ただし金額データ自体は保存しない（[finance_integration.md](finance_integration.md)）。

## 3. 画面遷移

```
                  ┌───────────────────────────────┐
                  │        下部タブバー            │
                  │   きょう ⇄ つながり            │
                  └───────────────────────────────┘
   きょう(ホーム) ────────────────── つながり
      │  │                             │
      │  ├─ 領域カードtap → 領域タイムライン(きょうの絞り込み)
      │  │        └─ Financeカードtap → Finance領域ビュー
      │  │                （さらに「アプリで開く」→ 既存家計アプリへディープリンク）
      │  │                             │
      │  └─ エントリtap ──────┐        ├─ ノードtap → エントリ詳細
      │                       ▼        │
      │                  エントリ詳細   ├─ ふりかえりカードtap → ふりかえり編集
      │                   （編集/削除/つなげる）
      │
      └─ FAB(+) → 記録シート（モーダル、どの画面からでも）
                       └─ のこす → 元の画面に戻る
```

- 下部タブは「きょう / つながり」の2つだけ。記録はタブにせず、右下の FAB からシートとして開く（UIMockUp 準拠）。
- 設定はきょう画面ヘッダーの歯車アイコンから。

## 4. しない機能（設計上の禁止事項）

要件定義書とConceptSheetで明示された「しないこと」。実装・改修時にうっかり足さないこと。

- 連続記録（ストリーク）表示、抜けた日を埋めさせる導線
- グラフ・達成率（きょう画面は「状態を眺める」画面。数値エントリも直近値の表示まで）
- リマインド・通知
- AI・スコアリング・KPI
- Finance データのローカル保存・編集機能
