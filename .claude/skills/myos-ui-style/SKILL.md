# MyOS UI スタイルガイド

ビジュアルの正は `MyOS_UIMockUp.pdf`、詳細仕様は `docs/ui/specification.md`。

## トーン

「管理ツール」ではなく「静かなノート」。クリーム地（`--color-bg`）+ テラコッタ（`--color-accent`）、大きめ角丸、淡い影。

## 実装ルール

1. 色・角丸・余白・フォントは `src/styles/tokens.css` の CSS 変数だけを使う。新しい色が要るときは先に tokens.css に追加する。
2. 領域チップ/ノードは `--domain-*` の8色。チップは淡い背景 + 濃い文字（ベタ塗り禁止）。
3. タップターゲット 44×44px 以上。セーフエリアは `env(safe-area-inset-*)`。
4. 保存成功のトーストは出さない（画面の変化で伝える）。トーストは失敗時のみ。
5. 確認ダイアログは削除とインポート（全置換）だけ。
6. アニメーションはシートの出入りと force layout の収束のみ。

## 文言

- やわらかい日本語:「のこす」「きょう」「つながり」「ふりかえり」。
- 命令形・督促形・感嘆符を使わない。「記録しましょう！」「まだ記録がありません」は禁止。空状態は静かに空のまま。

## 禁止パターン（提案もしない）

ストリーク / 達成率 / グラフ / バッジ / カウンタ / リマインド / 「さあ始めよう」系の空状態イラスト。
理由: 本アプリの目的は管理ではなく理解（requirements.md §2）。
